# Reviews by URL

Модуль Drupal 9/10/11 для управления отзывами клиентов с рейтингом и привязкой к страницам сайта по URL. Отзывы выводятся в виде стильных карточек с аватарами, звёздами рейтинга и Schema.org-разметкой для SEO.

## Возможности

### Управление отзывами
- **Кастомная сущность «Отзыв»** (`review_item`) с полями: имя автора, текст отзыва, рейтинг (1–5 звёзд), город, дата отзыва, URL-привязка, вес (сортировка), статус публикации
- **Админка с фильтрами** — поиск по имени/тексту, фильтр по URL и статусу публикации на странице `/admin/config/content/reviews-by-url`
- **Сортировка на фронтенде** — отзывы выводятся по дате (новые сверху); в админке — по весу для ручного управления порядком

### Привязка по URL
- **Точный путь** — `/catalog/metallocherepitsa` — отзыв появится только на этой странице
- **Маска пути** — `/catalog/*` — отзыв на всех страницах, начинающихся с `/catalog/`
- **Главная страница** — `<front>` — отзыв на главной независимо от её типа
- **Несколько URL** — один отзыв можно привязать к нескольким страницам
- **Псевдонимы URL** — автоматически разрешаются алиасы путей (system path ↔ URL alias)

### Вывод отзывов
- **Блок «Reviews by URL — Отзывы»** — размещается через *Структура → Схема блоков*, фильтрует отзывы по текущему URL страницы
- **Режим «Все отзывы»** — галочка в настройках блока; выводит ВСЕ отзывы с AJAX-пагинацией (12 на страницу) без изменения URL
- **Страница «Все отзывы»** — встроенный роут `/vse-otzyvy` с path-based пагинацией (`/vse-otzyvy/page/2`) — чистые URL без `?page`
- **Кнопка «Все отзывы»** — в заголовке блока ведёт на страницу со всеми отзывами (URL настраивается)

### Дизайн и оформление
- **Карточный дизайн** в стиле krovkomp.ru — белый фон, акцентный синий, золотые звёзды, Roboto
- **Двухколоночный layout** — Имя + Город слева, Рейтинг + Дата справа
- **Аватары с инициалами** — 6 цветовых вариантов, автоматически генерируются из имени автора
- **Border при hover** — обводка карточки появляется только при наведении
- **Адаптивность** — 3 колонки на десктопе, 1 на мобильных
- **Настраиваемые цвета** — 10+ цветовых параметров в настройках модуля без правки CSS
- **Минимальный шрифт 12px** — `max(12px, Xrem)` для соответствия требованиям поисковых систем

### SEO
- **Schema.org Review JSON-LD** — автоматическая генерация структурированных данных: `Product` + `AggregateRating` + массив `Review`
- **Дата публикации** — формат ISO 8601 (`YYYY-MM-DD`) в разметке
- **Заголовок h2** — наследует стили сайта (без принудительного font-size/color/font-weight)

### AJAX-пагинация (режим «Все отзывы»)
- Клик на номер страницы подгружает отзывы без перезагрузки
- URL страницы **не изменяется** — никакого `?page=2`
- Плавный скролл вверх блока при переключении страницы
- Обновление Schema.org-разметки после загрузки новых отзывов
- Визуальная обратная связь — полупрозрачность списка при загрузке

## Установка

### Через Composer (рекомендуется)

Модуль размещён на GitHub, а не на drupal.org, поэтому перед установкой нужно добавить репозиторий в `composer.json` вашего проекта:

**Шаг 1 — добавить репозиторий** (выполняется один раз):

```bash
composer config repositories.reviews_by_url vcs https://github.com/Mmitekk/rewiews_by_url
```

**Шаг 2 — установить модуль:**

```bash
composer require mmitekk/reviews_by_url
```

**Шаг 3 — включить модуль** на странице `/admin/modules` или через Drush:

```bash
drush en reviews_by_url -y
```

> **Почему нужен Шаг 1?** Модули с drupal.org ставятся сразу, потому что Drupal-проект уже содержит репозиторий `packages.drupal.org/8` в `composer.json`. Наш модуль — кастомный и хранится на GitHub, поэтому Composer нужно явно указать, где его искать. Это разовая операция — после добавления репозитория все будущие обновления (`composer update mmitekk/reviews_by_url`) будут работать без дополнительных настроек.

### Вручную

1. Скачайте архив с https://github.com/Mmitekk/rewiews_by_url
2. Распакуйте в `web/modules/custom/reviews_by_url/`
3. Включите модуль на странице `/admin/modules` или через Drush:
   ```bash
   drush en reviews_by_url -y
   ```

## Использование

### Добавление отзыва

1. Перейдите в *Конфигурация → Контент → Отзывы (Reviews by URL)* (`/admin/config/content/reviews-by-url`)
2. Нажмите **«Добавить отзыв»**
3. Заполните поля:
   - **Имя автора** — отображается в карточке отзыва
   - **Текст отзыва** — основной текст, поддерживает HTML-форматирование
   - **Рейтинг** — число от 1 до 5 (отображается звёздами)
   - **Город** — необязательное поле
   - **Дата отзыва** — если не указана, используется дата создания
   - **URL страниц** — один или несколько внутренних путей, на которых будет отображаться отзыв
   - **Вес** — порядок сортировки в админке (меньше = выше)
   - **Опубликован** — снятие галочки скрывает отзыв с сайта

### Размещение блока на странице

1. Перейдите в *Структура → Схема блоков*
2. Найдите блок **«Reviews by URL — Отзывы»**
3. Разместите его в нужном регионе
4. При размещении блока доступны настройки:
   - **«Показать все отзывы»** — если включено, блок выводит ВСЕ отзывы с AJAX-пагинацией (12 на страницу) без фильтрации по URL. Разместите такой блок на странице «Все отзывы»

### Страница «Все отзывы»

Для создания страницы со всеми отзывами есть два способа:

**Способ 1 — через блок (рекомендуется):**
1. Создайте страницу (Node или любую другую) по адресу, например `/vse-otzyvy`
2. Разместите блок **«Reviews by URL — Отзывы»** на эту страницу
3. В настройках блока поставьте галочку **«Показать все отзывы»**
4. Пагинация будет работать через AJAX — URL не изменяется

**Способ 2 — через встроенный роут:**
- Модуль автоматически регистрирует роут `/vse-otzyvy` с path-based пагинацией
- Страницы: `/vse-otzyvy`, `/vse-otzyvy/page/2`, `/vse-otzyvy/page/3` и т.д.
- URL остаются чистыми — никакого `?page` параметра

### Настройки модуля

Перейдите на вкладку **«Настройки»** (`/admin/config/content/reviews-by-url/settings`):

| Настройка | По умолчанию | Описание |
|-----------|--------------|----------|
| Заголовок блока | Отзывы | Заголовок секции отзывов |
| Сообщение при отсутствии | *(пусто)* | Текст, если отзывов нет на странице |
| URL страницы «Все отзывы» | `/vse-otzyvy` | URL для кнопки «Все отзывы» |
| Показывать рейтинг | Да | Звёзды рейтинга рядом с именем |
| Показывать дату | Да | Дата рядом с рейтингом |
| Показывать город | Да | Город под именем автора |
| Акцентный цвет | `#157fed` | Цвет кнопок, активного пейджера |
| Цвет имени автора | `#333333` | Цвет имени автора |
| Цвет текста отзыва | `#333333` | Цвет основного текста |
| Цвет разделителя | `#eeeeee` | Цвет линии между именем и текстом |
| Цвет даты | `#999999` | Цвет даты и города |
| Цвет звёзд | `#d89c11` | Золотой цвет звёзд рейтинга |
| Фон карточки | `#ffffff` | Белый фон карточки |
| Цвет обводки карточки | `#eeeeee` | Border при наведении на карточку |
| Фон секции | `#f2f2f2` | Серый фон всей секции |

## Привязка по URL

| Тип привязки | Пример | Результат |
|---|---|---|
| Точный путь | `/catalog/metallocherepitsa` | Отзыв только на этой странице |
| Маска пути | `/catalog/*` | Отзыв на всех страницах внутри `/catalog/` |
| Главная страница | `<front>` | Отзыв на главной странице |
| Несколько URL | `/about` + `/contacts` | Один отзыв на нескольких страницах |
| С алиасами | `/node/5` | Автоматически сопоставляется с URL-алиасом |
