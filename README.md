# Reviews by URL

Модуль Drupal 9/10/11 для добавления отзывов с рейтингом и привязкой их к страницам сайта по URL.

## Возможности

- **Сущность «Отзыв»** — кастомная Content Entity с полями: имя автора, текст отзыва, рейтинг (1–5 звёзд), город, дата отзыва, URL-привязка, вес (сортировка), статус публикации.
- **Привязка по URL** — каждый отзыв можно привязать к одной или нескольким страницам по внутреннему пути (например, `/catalog/metallocherepitsa`). Поддерживаются псевдонимы URL, маски путей (`/catalog/*`) и токен `<front>` для главной страницы.
- **Блок в Схеме блоков** — вывод отзывов через блок «Reviews by URL — Отзывы». Разместите его в нужном регионе через *Структура → Схема блоков*.
- **Карточный дизайн** — отзывы выводятся в виде карточек в стиле сайта krovkomp.ru: белый фон, акцентная синяя полоса, звёзды рейтинга, тени.
- **Schema.org Review** — автоматическая генерация JSON-LD разметки для SEO (тип `Product` с `AggregateRating` и массивом `Review`).
- **Настраиваемые цвета** — все визуальные цвета (акцент, текст, бордюр, звёзды, фон) можно изменить в настройках модуля без правки CSS.

## Установка

### Через Composer (рекомендуется)

Модуль размещён на GitHub, а не на drupal.org, поэтому перед установкой нужно добавить репозиторий в `composer.json` вашего проекта:

**Шаг 1 — добавить репозиторий** (выполняется один раз):

```bash
composer config repositories.reviews_by_url vcs https://github.com/Mmitekk/rewiews_by_url
```

**Шаг 2 — установить модуль:**

```bash
composer require mmitekk/reviews_by_url
```

**Шаг 3 — включить модуль** на странице `/admin/modules` или через Drush:

```bash
drush en reviews_by_url -y
```

> **Почему нужен Шаг 1?** Модули с drupal.org ставятся сразу, потому что Drupal-проект уже содержит репозиторий `packages.drupal.org/8` в `composer.json`. Наш модуль — кастомный и хранится на GitHub, поэтому Composer нужно явно указать, где его искать. Это разовая операция — после добавления репозитория все будущие обновления (`composer update mmitekk/reviews_by_url`) будут работать без дополнительных настроек.

### Вручную

1. Скачайте архив с https://github.com/Mmitekk/rewiews_by_url/archive/refs/tags/v1.0.0.zip
2. Распакуйте в `web/modules/custom/reviews_by_url/`.
3. Включите модуль на странице `/admin/modules` или через Drush:
   ```bash
   drush en reviews_by_url -y
   ```

## Использование

1. Перейдите в *Конфигурация → Контент → Отзывы (Reviews by URL)* (`/admin/config/content/reviews-by-url`).
2. Нажмите **«Добавить отзыв»**.
3. Заполните поля:
   - **Имя автора** — отображается как заголовок карточки.
   - **Текст отзыва** — основной текст, поддерживает HTML.
   - **Рейтинг** — число от 1 до 5 (звёзды).
   - **Город** — необязательное поле.
   - **Дата отзыва** — если не указана, используется дата создания.
   - **URL страниц** — один или несколько внутренних путей, на которых будет отображаться отзыв.
   - **Вес** — порядок сортировки (меньше = выше).
4. Перейдите в *Структура → Схема блоков* и разместите блок **«Reviews by URL — Отзывы»** в нужном регионе.
5. Настройте цвета и отображение на вкладке **«Настройки»** (`/admin/config/content/reviews-by-url/settings`).

## Настройки

| Настройка | По умолчанию | Описание |
|-----------|--------------|----------|
| Заголовок блока | Отзывы | Заголовок секции отзывов |
| Сообщение при отсутствии | *(пусто)* | Текст, если отзывов нет на странице |
| Показывать рейтинг | Да | Звёзды рейтинга рядом с именем |
| Показывать дату | Да | Дата под именем автора |
| Показывать город | Да | Город рядом с именем автора |
| Акцентный цвет | `#157fed` | Цвет заголовка, бордюра, ссылок |
| Цвет имени автора | `#333333` | Цвет имени автора |
| Цвет текста отзыва | `#333333` | Цвет основного текста |
| Цвет разделителя | `#eeeeee` | Цвет линии между именем и текстом |
| Цвет даты | `#999999` | Цвет даты и города |
| Цвет звёзд | `#d89c11` | Золотой цвет звёзд рейтинга |
| Фон карточки | `#ffffff` | Белый фон карточки |
| Цвет бордюра карточки | `#157fed` | Синяя акцентная линия |
| Фон секции | `#f2f2f2` | Серый фон всей секции |

## Привязка по URL

- **Точный путь**: `/catalog/metallocherepitsa` — отзыв появится только на этой странице.
- **Маска пути**: `/catalog/*` — отзыв появится на всех страницах, начинающихся с `/catalog/`.
- **Главная страница**: `<front>` — отзыв появится на главной странице независимо от её типа (нода, представление и т.д.).
- **Несколько URL**: один отзыв можно привязать к нескольким страницам, добавив несколько полей URL.
