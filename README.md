# Reviews by URL

Модуль Drupal 9/10/11 для управления отзывами клиентов с рейтингом и привязкой к страницам сайта по URL. Отзывы выводятся через настраиваемый блок с карточным дизайном, поддержкой Schema.org микроразметки и AJAX-пагинацией.

## Возможности

- **Сущность «Отзыв»** — кастомная Content Entity с полями: имя автора, текст отзыва (HTML), рейтинг (1–5 звёзд), город, дата отзыва, URL-привязка (множественное поле), вес (сортировка), статус публикации.
- **Привязка по URL** — каждый отзыв привязывается к одной или нескольким страницам по внутреннему пути. Поддерживаются псевдонимы URL, маски путей (`/catalog/*`) и токен `<front>` для главной страницы.
- **Блок «Reviews by URL»** — основной блок с фильтрацией отзывов по текущему URL страницы. Размещается через *Структура → Схема блоков*.
- **Режим «Все отзывы»** — галочка в настройках блока включает вывод всех отзывов с AJAX-пагинацией (по 12 на страницу). URL страницы при переключении страниц не меняется. Блок с этим режимом размещается на странице «Все отзывы».
- **Страница «Все отзывы»** — отдельный маршрут `/vse-otzyvy` с серверной пагинацией и чистыми URL (`/vse-otzyvy/page/2`). Заголовок и путь настраиваются.
- **Кнопка «Все отзывы»** — в заголовке блока отображается ссылка на страницу всех отзывов. URL подтягивается из настроек модуля. Кнопка автоматически скрывается, если пользователь уже находится на странице «Все отзывы».
- **Карточный дизайн** — отзывы выводятся в виде карточек: аватар с инициалами (6 цветов), имя и город слева, рейтинг и дата справа, текст отзыва ниже. Адаптивная сетка: 1 колонка на мобильных, 3 — на десктопе.
- **Schema.org Review** — автоматическая генерация JSON-LD разметки (тип `Product` с `AggregateRating` и массивом `Review`) для SEO.
- **Настраиваемые цвета** — 9 цветовых параметров (акцент, текст автора, текст отзыва, разделитель, дата, звёзды, фон карточки, бордюр карточки, фон секции) — всё меняется из админки без правки CSS.
- **Админка с фильтрами** — на странице `/admin/config/content/reviews-by-url` доступен поиск по имени/тексту, фильтрация по URL и статусу публикации, пагинация по 50 записей.
- **Сортировка** — на фронтенде отзывы сортируются по дате отзыва (новые сверху), в админке — по весу.

## Установка

### Через Composer (рекомендуется)

Модуль размещён на GitHub, а не на drupal.org, поэтому перед установкой нужно добавить репозиторий в `composer.json` вашего проекта:

**Шаг 1 — добавить репозиторий** (выполняется один раз):

```bash
composer config repositories.reviews_by_url vcs https://github.com/Mmitekk/rewiews_by_url
```

**Шаг 2 — установить модуль:**

```bash
composer require mmitekk/reviews_by_url
```

**Шаг 3 — включить модуль** на странице `/admin/modules` или через Drush:

```bash
drush en reviews_by_url -y
```

> **Почему нужен Шаг 1?** Модули с drupal.org ставятся сразу, потому что Drupal-проект уже содержит репозиторий `packages.drupal.org/8` в `composer.json`. Наш модуль — кастомный и хранится на GitHub, поэтому Composer нужно явно указать, где его искать. Это разовая операция — после добавления репозитория все будущие обновления (`composer update mmitekk/reviews_by_url`) будут работать без дополнительных настроек.

### Вручную

1. Скачайте архив с https://github.com/Mmitekk/rewiews_by_url
2. Распакуйте в `web/modules/custom/reviews_by_url/`.
3. Включите модуль на странице `/admin/modules` или через Drush:
   ```bash
   drush en reviews_by_url -y
   ```

## Использование

### Добавление отзывов

1. Перейдите в *Конфигурация → Контент → Отзывы (Reviews by URL)* (`/admin/config/content/reviews-by-url`).
2. Нажмите **«Добавить отзыв»**.
3. Заполните поля:
   - **Имя автора** — отображается как заголовок карточки.
   - **Текст отзыва** — основной текст, поддерживает HTML-форматирование.
   - **Рейтинг** — число от 1 до 5 (звёзды).
   - **Город** — необязательное поле, отображается рядом с именем.
   - **Дата отзыва** — если не указана, используется дата создания сущности.
   - **URL страниц** — один или несколько внутренних путей, на которых будет отображаться отзыв (подробнее в разделе «Привязка по URL»).
   - **Вес** — порядок сортировки в админке (меньше = выше).
   - **Опубликован** — чекбокс статуса.

### Размещение блока на странице

1. Перейдите в *Структура → Схема блоков* (`/admin/structure/block`).
2. Найдите регион, в котором хотите вывести отзывы, и нажмите **«Разместить блок»**.
3. Выберите **«Reviews by URL — Отзывы»**.
4. В настройках блока:
   - **Заголовок блока** — можно оставить пустым (заголовок берётся из настроек модуля).
   - **Показать все отзывы (режим «Все отзывы»)** — включите эту галочку, если блок размещается на странице «Все отзывы». В этом режиме блок выводит все отзывы с AJAX-пагинацией, без фильтрации по URL.

### Страница «Все отзывы»

Модуль автоматически регистрирует маршрут `/vse-otzyvy` для страницы со всеми отзывами. Путь можно изменить в настройках модуля (поле «URL страницы "Все отзывы"»).

Для работы страницы:

1. Создайте базовую страницу (Basic Page) с URL `/vse-otzyvy` (или любым другим).
2. Разместите блок **«Reviews by URL — Отзывы»** в нужном регионе, ограничив видимость только этой страницей.
3. В настройках блока включите галочку **«Показать все отзывы»**.

Альтернативный вариант — использовать встроенный маршрут модуля `/vse-otzyvy` без создания отдельной ноды. В этом случае контроллер модуля сам отрендерит страницу с отзывами и серверной пагинацией.

### Кнопка «Все отзывы»

В основном блоке (без галочки «Показать все отзывы») в заголовке рядом с названием отображается кнопка **«Все отзывы»**, ведущая на страницу со всеми отзывами. URL этой кнопки берётся из настроек модуля (поле «URL страницы "Все отзывы"»). Если поле оставлено пустым — кнопка не отображается. Кнопка автоматически скрывается, если пользователь уже находится на странице «Все отзывы» (включая страницы пагинации `/vse-otzyvy/page/2` и т.д.).

## Настройки

Перейдите в *Конфигурация → Контент → Отзывы → Настройки* (`/admin/config/content/reviews-by-url/settings`).

### Общие настройки

| Настройка | По умолчанию | Описание |
|-----------|--------------|----------|
| Заголовок блока | Отзывы | Заголовок секции отзывов. Отображается как h2 в блоке и h1 на странице «Все отзывы». |
| Сообщение при отсутствии | *(пусто)* | Текст, если отзывов нет на странице. Если пусто — блок полностью скрывается. |
| URL страницы «Все отзывы» | `/vse-otzyvy` | Внутренний путь для кнопки «Все отзывы». Оставьте пустым, чтобы скрыть кнопку. |
| Показывать рейтинг | Да | Звёзды рейтинга рядом с именем автора. |
| Показывать дату | Да | Дата отзыва под именем автора. |
| Показывать город | Да | Город рядом с именем автора. |

### Цвета и оформление

| Настройка | По умолчанию | Описание |
|-----------|--------------|----------|
| Акцентный цвет | `#157fed` | Цвет заголовка, аватара по умолчанию, ссылок. |
| Цвет имени автора | `#333333` | Цвет текста имени автора отзыва. |
| Цвет текста отзыва | `#333333` | Цвет основного текста отзыва. |
| Цвет разделителя | `#eeeeee` | Цвет линии между именем и текстом. |
| Цвет даты | `#999999` | Цвет даты и города. |
| Цвет звёзд | `#d89c11` | Золотой цвет звёзд рейтинга. |
| Фон карточки | `#ffffff` | Белый фон карточки отзыва. |
| Цвет бордюра карточки | `#eeeeee` | Цвет рамки карточки (видна при наведении). |
| Фон секции | `#f2f2f2` | Серый фон всей секции отзывов. |

Все цвета реализованы через CSS custom properties (`--review-color-*`) и применяются inline на обёртке блока, что позволяет легко переопределять их в теме.

## Привязка по URL

Каждый отзыв содержит множественное поле «URL страниц», куда можно добавить один или несколько внутренних путей сайта. Поддерживаются следующие форматы:

| Формат | Пример | Описание |
|--------|--------|----------|
| Точный путь | `/catalog/metallocherepitsa` | Отзыв появится только на этой странице |
| Путь без слеша | `catalog/metallocherepitsa` | Автоматически преобразуется в `/catalog/metallocherepitsa` |
| Маска пути | `/catalog/*` | Отзыв появится на всех страницах, начинающихся с `/catalog/` |
| Главная страница | `<front>` | Отзыв появится на главной странице (любой тип: нода, представление и т.д.) |

Модуль автоматически учитывает псевдонимы URL: если отзыв привязан к системному пути `/node/5`, а пользователь заходит по алиасу `/about`, отзыв всё равно отобразится.

## Schema.org

Модуль автоматически генерирует JSON-LD микроразметку на страницах с отзывами:

```json
{
  "@context": "https://schema.org",
  "@type": "Product",
  "name": "Название страницы",
  "aggregateRating": {
    "@type": "AggregateRating",
    "ratingValue": 4.7,
    "reviewCount": 15,
    "bestRating": 5,
    "worstRating": 1
  },
  "review": [
    {
      "@type": "Review",
      "author": { "@type": "Person", "name": "Анна К." },
      "reviewBody": "Отличный сервис...",
      "reviewRating": { "@type": "Rating", "ratingValue": 5, "bestRating": 5 },
      "datePublished": "2025-10-12"
    }
  ]
}
```

Микроразметка добавляется через JavaScript после загрузки страницы, что обеспечивает корректную работу с кэшированием Drupal.

## Админка

Список отзывов (`/admin/config/content/reviews-by-url`) содержит:

- **Поиск** — по имени автора или тексту отзыва (частичное совпадение).
- **Фильтр по URL** — показывает отзывы, привязанные к определённому пути.
- **Фильтр по статусу** — только опубликованные / только черновики / все.
- **Таблица** — колонки: ID, Автор, Рейтинг (звёзды), Город, URL, Статус, Вес, Обновлено, Операции (редактировать/удалить).
- **Пагинация** — по 50 записей на страницу.
- **Сброс фильтров** — кнопка «Сбросить» очищает все фильтры.

## Темизация

Модуль предоставляет два Twig-шаблона, которые можно переопределить в теме:

- **`reviews-by-url-block.html.twig`** — шаблон блока отзывов (используется как для обычного блока, так и для режима «Все отзывы»).
- **`reviews-by-url-all-page.html.twig`** — шаблон страницы «Все отзывы» (контроллер `/vse-otzyvy`).

Все визуальные параметры управляются через CSS custom properties, что позволяет стилизовать блок через CSS темы без переопределения шаблонов:

```css
.reviews-by-url-wrapper {
  --review-color-accent: #e53935;
  --review-color-star: #ffc107;
}
```
